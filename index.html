Fungsi dan Prinsip Kerja Protokol Komunikasi Jaringan

Topik: Model Protokol — HTTPS · TCP · Ethernet

1 HTTPS — HyperText Transfer Protocol Secure

Fungsi Utama: Layanan Web (Aman)

HTTPS adalah protokol lapisan aplikasi (OSI Layer 7) yang berfungsi sebagai fondasi komunikasi World Wide Web. HTTPS merupakan pengembangan dari HTTP dengan penambahan lapisan enkripsi TLS/SSL, sehingga seluruh data yang dikirim antara browser dan server tidak dapat dibaca oleh pihak ketiga di jaringan.[1,2]

Fungsi utama HTTPS dalam akses website adalah mengirim HTTP Request dari browser ke web server dan menerima HTTP Response (berupa halaman HTML, gambar, skrip, dsb.) secara terenkripsi. Protokol ini bekerja secara default pada port 443. Dibanding HTTP biasa (port 80), HTTPS memastikan tiga hal: kerahasiaan data (enkripsi), integritas data (tidak dimodifikasi di tengah jalan), dan autentikasi server (melalui sertifikat digital).[2,3]

Peran dalam Akses Website

Saat pengguna mengetik URL dengan awalan https://, browser akan: (1) membuka koneksi TCP ke server, (2) melakukan TLS Handshake untuk menyepakati kunci enkripsi, (3) mengirim HTTP Request (GET, POST, dll.) secara terenkripsi, lalu (4) menerima HTTP Response berisi konten halaman. Browser modern menampilkan ikon gembok sebagai tanda koneksi aman.[3,4]

Penjelasan Rangkaian Skematik Perangkat Keras (HTTPS)

Berikut perangkat keras yang terlibat saat HTTPS bekerja pada sisi klien (komputer pengguna):

Komponen HardwareFungsi dalam HTTPS
CPU (Processor)Menjalankan browser, memproses algoritma enkripsi TLS (AES, RSA, ECDHE), mengatur sesi HTTP. CPU modern umumnya memiliki instruksi AES-NI yang mempercepat enkripsi/dekripsi secara hardware.[6]
RAM (Memory)Menyimpan sesi TLS aktif, buffer data HTTP yang sedang diproses, dan sertifikat digital yang diunduh dari server. Diperlukan agar enkripsi/dekripsi berjalan cepat tanpa akses disk.[6]
NIC (Network Interface Card)Menerima dan mengirim paket-paket TCP/IP yang membawa data HTTPS terenkripsi. NIC beroperasi di Layer 2 (Data Link) dan Layer 1 (Physical), mengubah data digital menjadi sinyal listrik/optik di kabel Ethernet atau sinyal radio di Wi-Fi.[7]
Kabel / Media TransmisiKabel Ethernet (twisted pair Cat5e/Cat6) atau koneksi fiber optik menjadi media fisik di mana paket HTTPS mengalir sebagai sinyal listrik atau cahaya antar perangkat jaringan.[8]
Modem / RouterMeneruskan paket HTTPS ke internet melalui IP routing. Router memisahkan jaringan lokal (LAN) dari internet, sedangkan modem mengubah sinyal digital ke sinyal ISP (DSL, kabel, fiber).[8]
Web Server (sisi server)CPU server menjalankan software web (Apache, Nginx) dan memproses dekripsi TLS pada sisi server. Server juga memiliki NIC berkecepatan tinggi (10G/25G) untuk melayani banyak koneksi serentak.[6,7]

2 TCP — Transmission Control Protocol

Fungsi Utama: Pengiriman Andal (Reliable Delivery)

TCP adalah protokol lapisan transport (OSI Layer 4) yang menjamin pengiriman data secara andal, berurutan, dan bebas kesalahan antara dua host melalui jaringan IP. TCP merupakan tulang punggung sebagian besar komunikasi internet termasuk HTTP/HTTPS, email, dan transfer file.[9,10]

TCP adalah protokol connection-oriented — sebelum data dikirim, kedua pihak harus membangun koneksi terlebih dahulu melalui proses 3-Way Handshake. Setiap segmen data yang dikirim harus dikonfirmasi (ACK) oleh penerima; jika tidak ada konfirmasi, segmen akan dikirim ulang secara otomatis.[9,10]

Peran dalam Akses Website

Saat mengakses website, TCP bertugas menjamin semua data web tiba lengkap dan berurutan. Halaman HTML, gambar, dan skrip dikirim dalam banyak segmen TCP yang dapat tiba tidak berurutan karena melalui jalur jaringan yang berbeda. TCP yang bertugas menyusun ulang segmen-segmen tersebut, meminta pengiriman ulang jika ada yang hilang, dan menyerahkan data utuh ke lapisan aplikasi (browser).[9,11]

Penjelasan Rangkaian Skematik Perangkat Keras (TCP)

TCP adalah protokol software yang berjalan di dalam sistem operasi, namun perangkat keras berikut memainkan peran penting dalam implementasinya:

Komponen HardwareFungsi dalam TCP
CPU (Processor)Menjalankan TCP stack di dalam kernel sistem operasi (misalnya Linux TCP stack). CPU menghitung checksum segmen, mengelola nomor urut (sequence number), window size, dan mekanisme retransmission timer. Pada CPU modern, instruksi khusus offload checksum tersedia untuk meningkatkan performa.[6,7]
RAM (Memory)Menyimpan Transmission Control Block (TCB) — struktur data yang berisi tabel koneksi TCP, pointer buffer kirim/terima, antrian retransmisi, nomor urut saat ini, dan ukuran window. Setiap koneksi TCP aktif memiliki TCB sendiri di RAM.[12]
NIC (Network Interface Card)Pada NIC modern, fitur TCP Offload Engine (TOE) memungkinkan pemrosesan TCP dilakukan langsung di chip NIC, bukan di CPU utama. NIC juga mendukung Large Segment Offload (LSO) dan Receive Side Scaling (RSS) untuk mendistribusikan beban pemrosesan TCP ke banyak core CPU.[7]
PCIe Bus (Motherboard)NIC terhubung ke CPU dan RAM melalui slot PCIe (PCI Express) di motherboard. PCIe menyediakan jalur data berkecepatan tinggi (PCIe 3.0 x4 = ~16 Gbps) yang cukup untuk mendukung NIC 10GbE yang membawa banyak koneksi TCP serentak.[7]
Switch JaringanSwitch Ethernet di lapisan hardware meneruskan frame-frame TCP antar perangkat dalam LAN menggunakan tabel MAC address. Switch tidak membaca konten TCP, namun memastikan frame sampai ke tujuan yang benar di jaringan lokal.[8]

3 Ethernet — IEEE 802.3

Fungsi Utama: Jaringan Kabel Lokal (LAN Kabel)

Ethernet adalah teknologi jaringan kabel yang mendefinisikan standar komunikasi di lapisan Data Link (OSI Layer 2) dan Physical (OSI Layer 1). Ethernet adalah teknologi LAN (Local Area Network) yang paling dominan digunakan saat ini, beroperasi pada kecepatan 10 Mbps hingga lebih dari 100 Gbps, dan distandarkan sebagai IEEE 802.3.[13,14]

Ethernet mengemas data menjadi unit-unit yang disebut frame. Setiap frame berisi alamat MAC tujuan dan sumber, data yang dikirim, dan kode pendeteksi kesalahan (CRC/FCS). Frame dikirim melalui media fisik (kabel twisted pair atau fiber optik) dan disampaikan ke perangkat yang tepat berdasarkan MAC address.[13,14,15]

Peran dalam Akses Website

Ethernet bertugas mengirim frame-frame data dalam jaringan lokal. Ketika paket HTTPS dikirim dari komputer pengguna menuju router/gateway internet, Ethernet yang mengangkutnya sebagai frame dari satu perangkat ke perangkat berikutnya dalam LAN. Ethernet tidak peduli dengan konten data (apakah itu HTTPS, TCP, atau lainnya) — tugasnya hanya memastikan frame tiba di MAC address tujuan yang benar dalam segmen jaringan lokal.[13,15,16]

Penjelasan Rangkaian Skematik Perangkat Keras (Ethernet)

Ethernet adalah teknologi yang paling erat berkaitan dengan perangkat keras fisik. Berikut komponen hardware yang terlibat:

Komponen HardwareFungsi dalam Ethernet
NIC — Network Interface CardKomponen inti Ethernet pada setiap perangkat. NIC memiliki controller (mini-CPU) yang memproses data masuk/keluar, memiliki port RJ-45 untuk kabel Ethernet, dan memiliki alamat MAC unik 48-bit yang di-burn ke chip ROM. NIC mengubah data digital menjadi sinyal listrik untuk dikirim ke kabel dan sebaliknya.[7,8]
MAC Address ROM (di NIC)Sebuah chip ROM (Read-Only Memory) di NIC yang menyimpan alamat MAC (Media Access Control) secara permanen. MAC address 48-bit ini digunakan Ethernet untuk mengidentifikasi setiap perangkat secara unik dalam jaringan lokal dan menentukan tujuan pengiriman frame.[7,17]
PHY Chip (Physical Layer Chip)Chip khusus di NIC yang bertugas mengubah sinyal digital (bit 0 dan 1) menjadi sinyal analog/listrik yang bisa dikirim melalui kabel Ethernet, dan sebaliknya. PHY chip menangani encoding (Manchester, NRZ, PAM-4) sesuai standar kecepatan Ethernet (10M/100M/1G/10G).[7]
Kabel Twisted Pair (UTP/STP)Media transmisi fisik Ethernet. Kabel Cat5e mendukung Gigabit Ethernet (1000BASE-T), kabel Cat6/Cat6a mendukung 10GBase-T. Setiap pasang kawat terpilin untuk mengurangi interferensi elektromagnetik (crosstalk). Terhubung ke NIC dan switch menggunakan konektor RJ-45.[8]
Switch Ethernet (L2 Switch)Perangkat jaringan yang meneruskan frame Ethernet antar port berdasarkan tabel MAC address. Switch mempelajari MAC address dari frame yang masuk, menyimpannya di tabel CAM (Content Addressable Memory), dan meneruskan frame hanya ke port tujuan yang benar — bukan broadcast ke semua port.[8,16]
Buffer / RAM di SwitchSwitch memiliki RAM internal untuk menyimpan frame sementara (store-and-forward) sebelum diteruskan ke port tujuan. Buffer ini penting saat port tujuan sedang sibuk dan frame harus menunggu sebelum dikirim.
Kristal Osilator (Clock)NIC dan switch memiliki kristal osilator yang menghasilkan sinyal clock. Preamble 7-byte di setiap frame Ethernet digunakan untuk sinkronisasi clock antara pengirim dan penerima, memastikan keduanya membaca bit pada waktu yang tepat.[14,15]
